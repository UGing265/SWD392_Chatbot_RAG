# File Processing và AI Integration cho Chatbot Học liệu

---

## 1. Tổng quan

Tài liệu này mô tả chi tiết về cách xử lý tài liệu đầu vào (PDF/DOCX/PPT/PPTX), embedding, vector search, chatbot RAG flow, và tích hợp AI trong ứng dụng C# (.NET). Mục tiêu là triển khai pipeline end-to-end từ upload file đến trả lời câu hỏi có trích dẫn nguồn.

## 2. Pipeline tổng thể

```
[Upload] → [Store File] → [Extract Text] → [Chunk] → [Embed] → [Vector Store]
                                                                      ↓
                                                               [Chat Query] → [Retrieve] → [Prompt Build] → [LLM] → [Answer + Citations]
```

## 3. Xử lý file (File Handling)

### 3.1. Upload và lưu trữ

#### Yêu cầu upload
- File types: PDF, DOCX, PPT, PPTX
- Max size: 50MB (có thể config)
- Chỉ người dùng đã xác thực mới upload

#### Lưu trữ
- **Demo local**: `wwwroot/uploads/{CourseId}/{Guid}.{ext}`
- **Production**: Azure Blob Storage / AWS S3 / MinIO
- Lưu metadata vào SQL Server (Document table)

#### Validation
```csharp
public class FileValidator
{
    private static readonly string[] AllowedExtensions = { ".pdf", ".docx", ".pptx", ".ppt" };
    private const long MaxSize = 50 * 1024 * 1024; // 50MB

    public async Task<ValidationResult> ValidateAsync(IFormFile file)
    {
        var ext = Path.GetExtension(file.FileName).ToLowerInvariant();
        if (!AllowedExtensions.Contains(ext))
            return ValidationResult.Failed("Chỉ chấp nhận file PDF, DOCX, PPTX.");

        if (file.Length > MaxSize)
            return ValidationResult.Failed("File vượt quá 50MB.");

        return ValidationResult.Success();
    }
}
```

## 4. Extract text từ file

### 4.1. PDF (text-based)

**Library đề xuất:** `PdfSharpCore` (open source), `iTextSharp`, `PdfPig`, `IronPDF` (commercial).

```csharp
public interface IPdfTextExtractor
{
    Task<ExtractedDocument> ExtractAsync(string filePath);
}

public class PdfPigExtractor : IPdfTextExtractor
{
    public async Task<ExtractedDocument> ExtractAsync(string filePath)
    {
        using var stream = File.OpenRead(filePath);
        var pdf = PdfDocument.Open(stream);
        var pages = new List<PageText>();

        for (int i = 0; i < pdf.PageCount; i++)
        {
            var page = pdf.GetPage(i + 1);
            var text = page.Text;
            pages.Add(new PageText { PageNumber = i + 1, Text = text });
        }

        return new ExtractedDocument
        {
            Pages = pages,
            Metadata = new DocumentMetadata
            {
                PageCount = pdf.PageCount,
                Title = pdf.DocumentInformation?.Title
            }
        };
    }
}
```

    {
        using var ppt = PresentationDocument.Open(filePath, false);
        var slides = ppt.PresentationPart?.SlideParts;
        var pageList = new List<PageText>();

        int idx = 1;
        foreach (var slidePart in slides ?? Enumerable.Empty<SlidePart>())

        }

    }
}

## 5. Chunking strategies

Mục tiêu: chia văn bản dài thành các đoạn nhỏ để đưa vào embedding.

### 5.1. Fixed-size chunking (token-based)

- Cắt theo số token cố định (300/500/800 tokens).
- Có overlap (10-20%) để giữ continuity.

**Library:** `Microsoft.ML.Tokenizers` (BPE, tách theo token).

```csharp
public interface IChunker
{
    Task<IReadOnlyList<Chunk>> ChunkAsync(ExtractedDocument doc, ChunkingConfig config);
}

public class FixedSizeChunker : IChunker
{
    private readonly Tokenizer _tokenizer;

    public FixedSizeChunker()
    {
        _tokenizer = Tokenizer.Create();
    }

    public async Task<IReadOnlyList<Chunk>> ChunkAsync(ExtractedDocument doc, ChunkingConfig config)
    {
        var chunks = new List<Chunk>();
        int overlapTokens = (int)(config.ChunkSize * config.OverlapRatio);
        int globalIndex = 0;

        foreach (var page in doc.Pages)
        {
            var tokens = _tokenizer.Encode(page.Text).Select(t => t.Value).ToList();
            int start = 0;
            while (start < tokens.Count)
            {
                int chunkSize = Math.Min(config.ChunkSize, tokens.Count - start);
                var chunkTokens = tokens.GetRange(start, chunkSize);
                var chunkText = _tokenizer.Decode(chunkTokens.Select(t => new Token(t)).ToList());

                chunks.Add(new Chunk
                {
                    ChunkId = Guid.NewGuid(),
                    DocumentPage = page.PageNumber,
                    ChunkIndex = globalIndex++,
                    Text = chunkText,
                    StartToken = start,
                    EndToken = start + chunkSize
                });

                start += chunkSize - overlapTokens;
            }
        }

        return chunks;
    }
}
```

### 5.2. Structure-based chunking

Chia theo heading/slide boundaries (nếu tài liệu có cấu trúc).

```csharp
public class StructureBasedChunker : IChunker
{
    public Task<IReadOnlyList<Chunk>> ChunkAsync(ExtractedDocument doc, ChunkingConfig config)
    {
        var chunks = new List<Chunk>();
        var headingRegex = new Regex(@"^(#+\s+)|^(\d+\.\s+)");

        foreach (var page in doc.Pages)
        {
            var lines = page.Text.Split('\n');
            var currentChunk = new StringBuilder();
            int chunkIndex = 0;

            foreach (var line in lines)
            {
                if (headingRegex.IsMatch(line.Trim()))
                {
                    // Save previous chunk if non-empty
                    if (currentChunk.Length > 0)
                    {
                        chunks.Add(new Chunk
                        {
                            ChunkId = Guid.NewGuid(),
                            DocumentPage = page.PageNumber,
                            ChunkIndex = chunkIndex++,
                            Text = currentChunk.ToString()
                        });
                        currentChunk.Clear();
                    }
                }
                currentChunk.AppendLine(line);
            }

            if (currentChunk.Length > 0)
            {
                chunks.Add(new Chunk { ChunkId = Guid.NewGuid(), DocumentPage = page.PageNumber, ChunkIndex = chunkIndex++, Text = currentChunk.ToString() });
            }
        }

        return Task.FromResult((IReadOnlyList<Chunk>)chunks);
    }
}
```

### 5.3. Hybrid chunking

Kết hợp cả hai: ưu tiên split theo heading, nhưng giới hạn chunk size.

## 6. Embedding

### 6.1. Lựa chọn embedding model

| Model | Language | Size | Nơi chạy |
|-------|----------|------|----------|
| multilingual-e5-base | đa ngôn ngữ | 550M | local hoặc HuggingFace |
| PhoBERT-base | tiếng Việt | 135M | local |
| bge-m3 | đa ngôn ngữ | 560M | local |
| text-embedding-3-small | đa ngôn ngữ | cloud | OpenAI API |

### 6.2. Gọi embedding từ .NET

#### Trường hợp local model (ONNX / transformers.NET)

**Library:** `Microsoft.ML.OnnxRuntime`, `Microsoft.SemanticKernel` (có embedding provider)

```csharp
using Microsoft.ML.OnnxRuntime;
using Microsoft.ML.OnnxRuntime.Tensors;

public class OnnxEmbeddingProvider : IEmbeddingProvider
{
    private readonly InferenceSession _session;
    private readonly Tokenizer _tokenizer;
    private const int EmbeddingDimension = 768;

    public async Task<float[]> GetEmbeddingAsync(string text)
    {
        var tokens = _tokenizer.Encode(text);
        var inputIds = tokens.Select(t => (long)t.Value).Take(512).ToArray();
        var attentionMask = new long[512];
        for (int i = 0; i < inputIds.Length; i++) attentionMask[i] = 1;

        var container = new List<NamedOnnxValue>
        {
            NamedOnnxValue.CreateFromTensor("input_ids",
                new DenseTensor<long>(new[] { 1, 512 }, inputIds)),
            NamedOnnxValue.CreateFromTensor("attention_mask",
                new DenseTensor<long>(new[] { 1, 512 }, attentionMask))
        };

        using var results = _session.Run(container);
        var embedding = results.First().AsEnumerable<float>().ToArray();
        // Normalize L2
        var norm = Math.Sqrt(embedding.Sum(x => x * x));
        for (int i = 0; i < embedding.Length; i++) embedding[i] /= (float)norm;

        return embedding;
    }
}
```

#### Trường hợp OpenAI embedding API

```csharp
public class OpenAIEmbeddingProvider : IEmbeddingProvider
{

    private readonly string _apiKey;

    public async Task<float[]> GetEmbeddingAsync(string text)
    {

```

## 7. Vector Store

### 7.1. Tại sao cần vector store?

Vector store cho phép tìm kiếm top-K chunk có embedding gần với query embedding nhất (similarity search).

### 7.2. Lựa chọn

| Option | Pros | Cons |
|--------|------|------|
| Qdrant | REST API, persistence, scalable | Cần chạy service riêng |
| Chroma | Nhẹ, embedded mode | Ít tính năng enterprise |
| FAISS | Nhẹ, nhanh | Không có persistence sẵn, ph tự quản |
| SQL Server + pgvector-like | Cùng SQL Server, đơn giản | Cần cài extension, performance phải benchmark |

### 7.3. Gợi ý: Dùng Qdrant

**Cài đặt:** `docker run -p 6333:6333 qdrant/qdrant`

**C# client:** `Qdrant.Client`

```csharp
using Qdrant.Client;
using Qdrant.Client.Grpc;

public class QdrantVectorStore : IVectorStore
{
    private readonly QdrantClient _client;
    private const string CollectionName = "chunks";

    public async Task EnsureCollectionAsync(int vectorSize = 768)
    {
        await _client.CreateCollectionAsync(CollectionName, new VectorParams
        {
            Size = (uint)vectorSize,
            Distance = Distance.Cosine
        });
    }

    public async Task UpsertChunksAsync(IReadOnlyList<Chunk> chunks, float[][] embeddings)
    {
        var points = chunks.Select((chunk, idx) => new PointStruct
        {
            Id = (ulong)chunk.ChunkId.GetHashCode(),
            Vector = new Vector { Data = { embeddings[idx].ToList() } },
            Payload =
            {
                ["document_id"] = chunk.DocumentId.ToString(),
                ["chunk_index"] = chunk.ChunkIndex,
                ["page_number"] = chunk.DocumentPage,
                ["text"] = chunk.Text
            }
        }).ToList();

        await _client.UpsertAsync(CollectionName, points);
    }

    public async Task<IReadOnlyList<RetrievedChunk>> SearchAsync(float[] queryEmbedding, int topK)
    {
        var searchResult = await _client.SearchAsync(CollectionName, queryEmbedding, limit: (ulong)topK);
        return searchResult.Select(r => new RetrievedChunk
        {
            Score = (float)r.Score,
            Chunk = new Chunk
            {
                ChunkId = Guid.Parse(r.Id.ToString()),
                DocumentId = Guid.Parse(r.Payload["document_id"].StringValue),
                ChunkIndex = r.Payload["chunk_index"].IntegerValue,
                DocumentPage = (int)r.Payload["page_number"].IntegerValue,
                Text = r.Payload["text"].StringValue
            }
        }).ToList();
    }
}
```

### 7.4. Tách metadata lưu SQL Server, chỉ lưu vector trong vector store

**SQL Server lưu:**
- Document metadata
- Chunk metadata (chứa thông tin về document, vị trí, nhưng không chứa embedding vector)

**Vector store lưu:**
- ChunkId làm primary key
- Embedding vector
- Minimal payload (document_id, chunk_index, page_number, text) để trả về khi search

## 8. Chatbot RAG Flow

### 8.1. Pipeline chi tiết

```
User Query
    ↓
Query Rewrite (optional: cleaner, more retrieval-friendly)
    ↓
Get Query Embedding
    ↓
Vector Store Search → top-K chunks (cosine similarity)
    ↓
Filter by similarity threshold (e.g., 0.6)
    ↓
Rerank (optional cross-encoder)
    ↓
Build Prompt:
  System: "Bạn là trợ lý dạy học. Chỉ trả lời dựa trên context đã cho."
  Context: [Các chunk retrieved]
  Chat History: [N hội thoại gần nhất]
  Question: [Câu hỏi]
    ↓
Call LLM API (OpenAI GPT-4o-mini, Claude 3.5 Sonnet, hoặc local Llama)
    ↓
Parse Answer → extract citations (mapping với chunk ids)
    ↓
Return Answer + Citations
```

### 8.2. Prompt Template mẫu

```text
Hệ thống:
Bạn là trợ lý học tập. Chỉ trả lời dựa trên tài liệu được cung cấp dưới đây. Nếu không có thông tin, hãy trả lời "Không tìm thấy thông tin trong tài liệu." Không được bịa đặt.

Tài liệu tham khảo:
{context}

Lịch sử trò chuyện:
{chat_history}

Câu hỏi: {question}

Trả lời:
```

### 8.3. Code ChatService skeleton

```csharp
public class ChatService : IChatService
{
    private readonly IVectorStore _vectorStore;
    private readonly IEmbeddingProvider _embeddingProvider;
    private readonly ILLMProvider _llmProvider;
    private readonly IChatRepository _chatRepo;

    public async Task<ChatResponse> ProcessMessageAsync(ChatSession session, string question)
    {
        // 1. Retrieve
        var queryEmbedding = await _embeddingProvider.GetEmbeddingAsync(question);
        var retrieved = await _vectorStore.SearchAsync(queryEmbedding, topK: 5);
        var filtered = retrieved.Where(x => x.Score >= 0.6).ToList();

        if (filtered.Count == 0)
        {
            return new ChatResponse
            {
                Answer = "Không tìm thấy thông tin trong tài liệu.",
                Citations = new List<Citation>()
            };
        }

        // 2. Build prompt
        var context = string.Join("\n---\n", filtered.Select(c => c.Chunk.Text));
        var history = await _chatRepo.GetLastNMessagesAsync(session.SessionId, 10);
        var prompt = PromptBuilder.BuildChatPrompt(context, history, question);

        // 3. Call LLM
        var answer = await _llmProvider.GenerateAsync(prompt);

        // 4. Extract citations
        var citations = CitationExtractor.ExtractFromAnswer(answer, filtered.Select(r => r.Chunk).ToList());

        // 5. Save message
        await _chatRepo.AddMessageAsync(new ChatMessage
        {
            SessionId = session.SessionId,
            Role = MessageRole.User,
            Content = question,
            CreatedAt = DateTime.UtcNow
        });
        await _chatRepo.AddMessageAsync(new ChatMessage
        {
            SessionId = session.SessionId,
            Role = MessageRole.Assistant,
            Content = answer,
            Citations = JsonSerializer.Serialize(citations),
            PromptTokens = prompt.TokenCount,
            CompletionTokens = answer.TokenCount,
            LatencyMs = prompt.Latency + answer.Latency,
            CreatedAt = DateTime.UtcNow
        });

        return new ChatResponse { Answer = answer, Citations = citations };
    }
}
```

## 9. Citations và traceability

### 9.1. Làm sao extract citation từ câu trả lời?

Có 2 cách:

**Cách 1: LLM sinh theo format JSON** – thêm vào prompt yêu cầu trả về JSON với `citations` chứa chunk indices.

**Cách 2: Post-parse** – tìm các chunk text xuất hiện trong câu trả lời, map về chunk metadata.

```csharp
public static class CitationExtractor
{
    public static IReadOnlyList<Citation> ExtractFromAnswer(string answer, IReadOnlyList<Chunk> usedChunks)
    {
        var citations = new List<Citation>();

        foreach (var chunk in usedChunks)
        {
            // Nếu câu trả lời chứa một phần của chunk text, coi là citation
            if (answer.Contains(chunk.Text.Substring(0, Math.Min(100, chunk.Text.Length)), StringComparison.OrdinalIgnoreCase))
            {
                citations.Add(new Citation
                {
                    ChunkId = chunk.ChunkId,
                    DocumentId = chunk.DocumentId,
                    PageNumber = chunk.DocumentPage,
                    Snippet = chunk.Text.Substring(0, Math.Min(200, chunk.Text.Length))
                });
            }
        }

        return citations;
    }
}
```

## 10. Guardrails chống hallucination

### 10.1. System prompt mạnh

Đã đề cập ở trên: ràng buộc LLM chỉ trả lời theo context.

### 10.2. Similarity threshold

Nếu top-K max similarity < 0.6 → trả về `no info`.

### 10.3. RAGAS metrics

- **Faithfulness**: Câu trả lời có trung thực với context không?
- **Answer Relevance**: Câu trả lời có liên quan với câu hỏi không?
- **Context Recall**: Context retrieved có bao phủ thông tin cần thiết không?

## 15. Troubleshooting

### Problem: OCR không trích xuất được text rõ ràng
- Kiểm tra chất lượng PDF scan.
- Thử các engine OCR khác (Tesseract vs IronOcr).
- Dùng preprocessing: deskew, denoise.

### Problem: Retrieved chunk không liên quan
- Kiểm tra embedding model có phù hợp tiếng Việt không.
- Thử chunk size khác (nhỏ hơn/hơn).
- Có thể cần rerank bằng cross-encoder.

### Problem: LLM trả lời hallucinate
- Tăng system prompt mạnh hơn.
- Dùng retrieval-augmented generation với context rõ ràng.
- Kiểm tra RAGAS faithfulness score để đánh giá.
